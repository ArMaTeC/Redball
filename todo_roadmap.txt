**Implementation Details**:

**Security Features**:
- State parameter validation to prevent CSRF attacks
- PKCE (Proof Key for Code Exchange) for secure public clients
- Timeout handling for callback (120 seconds default)
- Secure random generation for code_verifier and state

**Test Coverage**: SSOServiceOIDCTests.cs with PKCE generation, token deserialization, session validation tests

- Implemented `ExtractMsiAsync()` using msiexec administrative install (/a /qn)
- Implemented `ExtractZipAsync()` with progress tracking per entry
- Added `VerifyPackageHashAsync()` for SHA-256 hash verification
- Added proper error handling and logging throughout

**Implementation Details**:
1. **MSI Extraction**: Downloads to temp, runs `msiexec /a "file.msi" /qn TARGETDIR="path"`
2. **ZIP Extraction**: Downloads to temp, uses `System.IO.Compression.ZipFile` with entry-by-entry progress
3. **Hash Verification**: SHA-256 hash of main executable compared to expected hash
4. **Progress Reporting**: 0-100% with download (0-30%) and extraction (30-100%) phases
5. **Cleanup**: Temp files deleted in finally blocks

**Security Features**:
- Hash verification before accepting extracted package
- Temp file cleanup even on failure
- Error logging without exposing sensitive paths

**Dependencies**: System.IO.Compression (ZipFile), System.Net.Http

**Test Coverage**: BlueGreenUpdateServiceTests.cs with ZIP extraction, hash verification, result type tests
