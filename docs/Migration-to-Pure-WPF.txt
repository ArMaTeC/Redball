OVERVIEW
--------

Redball.UI.WPF C# project. This eliminates the IPC layer, reduces failure
points, and produces a single self-contained .exe.

  - ConfigService.cs      (config load/save - DONE)
  - HotkeyService.cs      (global hotkeys - DONE)

  - SettingsWindow.xaml/.cs (settings UI - DONE)
  - AboutWindow.xaml/.cs  (about dialog - DONE)
  - MainViewModel.cs      (view model - DONE)

PHASE 1: CORE KEEP-AWAKE ENGINE (Priority: CRITICAL)
================================================================================
This is the heart of Redball. Without it, nothing works.

Step 1.1: Create KeepAwakeService.cs
  Location: src/Redball.UI.WPF/Services/KeepAwakeService.cs
  Port from: Set-KeepAwakeState, Send-HeartbeatKey, Set-ActiveState,
             Switch-ActiveState, Start-TimedAwake
  
  Implementation:
  - P/Invoke for SetThreadExecutionState (kernel32.dll)
    - ES_CONTINUOUS        = 0x80000000
    - ES_SYSTEM_REQUIRED   = 0x00000001
    - ES_DISPLAY_REQUIRED  = 0x00000002
  - P/Invoke for SendInput (user32.dll) for F15 heartbeat
  - DispatcherTimer for periodic heartbeat (config.HeartbeatSeconds interval)
  - DispatcherTimer for duration checking (1-second interval)
  - Properties: IsActive, PreventDisplaySleep, UseHeartbeat, Until (DateTime?)
  - Methods: SetActive(bool, DateTime?), Toggle(), StartTimed(int minutes)
  - Events: ActiveStateChanged, TimedAwakeExpired
  
  Key Design:
  ```csharp
  public class KeepAwakeService : IDisposable
  {
      [DllImport("kernel32.dll")]
      static extern uint SetThreadExecutionState(uint esFlags);
      
      private DispatcherTimer _heartbeatTimer;
      private DispatcherTimer _durationTimer;
      
      public bool IsActive { get; private set; }
      public bool PreventDisplaySleep { get; set; }
      public bool UseHeartbeat { get; set; }
      public DateTime? Until { get; private set; }
      
      public event EventHandler<bool>? ActiveStateChanged;
      
      public void SetActive(bool active, DateTime? until = null) { ... }
      public void Toggle() => SetActive(!IsActive);
      public void StartTimed(int minutes) => SetActive(true, DateTime.Now.AddMinutes(minutes));
  }
  ```

Step 1.2: Remove IPC layer
  - DELETE: src/Redball.UI.WPF/IPC/ folder
  - DELETE: src/Redball.UI.WPF/Services/IpcClientService.cs
  - UPDATE: MainViewModel.cs - replace IPC calls with direct KeepAwakeService calls
  - UPDATE: App.xaml.cs - remove StartIpcServerAsync, pipe server code, IpcRequest/IpcResponse
  - UPDATE: Redball.UI.WPF.csproj - remove System.IO.Pipes package reference

Step 1.3: Wire KeepAwakeService into MainViewModel
  - Inject KeepAwakeService into MainViewModel
  - ToggleActiveCommand calls service.Toggle() directly
  - Bind UI state to service.IsActive
  - Update tray icon based on service state (active/timed/paused)

VERIFICATION: After Phase 1, the app should keep the system awake when active
and release when paused, with periodic heartbeat - all without PowerShell.

PHASE 2: MONITORING SERVICES (Priority: HIGH)
================================================================================
Auto-pause/resume features that react to system conditions.

Step 2.1: Create BatteryMonitorService.cs
  Location: src/Redball.UI.WPF/Services/BatteryMonitorService.cs
  Port from: Get-BatteryStatus, Test-BatteryThreshold, Update-BatteryAwareState
  
  Implementation:
  - Use System.Windows.Forms.SystemInformation.PowerStatus (or WMI via
    System.Management) for battery info
  - Cache results for 60 seconds (like PS version)
  - Properties: HasBattery, IsOnBattery, ChargePercent, IsEnabled
  - Methods: CheckAndAutoPause(KeepAwakeService)
  - Called by the duration timer every second when enabled

Step 2.2: Create NetworkMonitorService.cs
  Location: src/Redball.UI.WPF/Services/NetworkMonitorService.cs
  Port from: Get-NetworkStatus, Update-NetworkAwareState
  
  Implementation:
  - Use System.Net.NetworkInformation.NetworkInterface for adapter status
  - Properties: IsConnected, AdapterName, IsEnabled
  - Methods: CheckAndAutoPause(KeepAwakeService)
  - Alternative: subscribe to NetworkChange.NetworkAvailabilityChanged event

Step 2.3: Create IdleDetectionService.cs
  Location: src/Redball.UI.WPF/Services/IdleDetectionService.cs
  Port from: Get-IdleTimeMinute, Update-IdleAwareState (uses GetLastInputInfo)
  
  Implementation:
  - P/Invoke for GetLastInputInfo (user32.dll)
  - struct LASTINPUTINFO { uint cbSize; uint dwTime; }
  - Properties: IdleMinutes, ThresholdMinutes, IsEnabled
  - Methods: CheckAndAutoPause(KeepAwakeService)

Step 2.4: Create ScheduleService.cs
  Location: src/Redball.UI.WPF/Services/ScheduleService.cs
  Port from: Test-ScheduleActive, Update-ScheduleState
  
  Implementation:
  - Check current time against config ScheduleStartTime/ScheduleStopTime
  - Check current day against config ScheduleDays list
  - Properties: IsInSchedule, IsEnabled
  - Methods: CheckAndAutoPause(KeepAwakeService)

Step 2.5: Create PresentationModeService.cs
  Location: src/Redball.UI.WPF/Services/PresentationModeService.cs
  Port from: Test-PresentationMode, Update-PresentationModeState
  
  Implementation:
  - Check for running processes: POWERPNT, Teams (with screen sharing)
  - Check registry: HKCU\Software\Microsoft\MobilePC\AdaptableSettings
  - Cache results for 10 seconds
  - Properties: IsPresenting, Source, IsEnabled
  - Methods: CheckAndAutoActivate(KeepAwakeService)

Step 2.6: Integrate monitors into duration timer
  The existing KeepAwakeService._durationTimer (1s tick) calls each monitor:
  ```csharp
  private void OnDurationTimerTick(object? sender, EventArgs e)
  {
      if (Config.BatteryAware) _batteryMonitor.CheckAndAutoPause(_keepAwake);
      if (Config.NetworkAware) _networkMonitor.CheckAndAutoPause(_keepAwake);
      if (Config.IdleDetection) _idleDetection.CheckAndAutoPause(_keepAwake);
      if (Config.ScheduleEnabled) _schedule.CheckAndAutoPause(_keepAwake);
      if (Config.PresentationMode) _presentation.CheckAndAutoActivate(_keepAwake);
      CheckTimedExpiry();
  }
  ```

VERIFICATION: Enable each monitor in settings, simulate conditions (unplug
battery, disconnect network, go idle), verify auto-pause/resume works.

================================================================================
RISKS & MITIGATIONS
================================================================================

1. SetThreadExecutionState thread affinity
   Risk: Must be called from same thread repeatedly
   Mitigation: Use DispatcherTimer (runs on UI thread) like PS version

2. Battery WMI queries may be slow
   Risk: CIM/WMI calls can block UI
   Mitigation: Run in background Task, cache results for 60s

3. Singleton mutex across PS and WPF
   Risk: Old PS instance blocks new WPF instance
   Mitigation: Use same mutex name, detect and offer to kill old process

4. Config file format compatibility
   Risk: PS and C# read same Redball.json
   Mitigation: RedballConfig class already matches PS config schema exactly

5. TypeThing hotkey registration
   Risk: Already complex in PS, needs careful port
   Mitigation: HotkeyService.cs already handles this - reuse existing code
