## Issue 2: Duplicate Tray Icons (HIGH PRIORITY)

**Symptom:** Two empty tray icon spaces appear instead of one

**Log Evidence:**
```
[2026-03-11 15:40:00] === SetupTrayIcon called ===  (first call)
[2026-03-11 15:40:00] Tray icon visibility set to Visible
[2026-03-11 15:40:00] === SetupTrayIcon complete ===
[2026-03-11 15:40:00] MainWindow loaded, hiding window...
[2026-03-11 15:40:00] === SetupTrayIcon called ===  (second call!)
[2026-03-11 15:40:00] Tray icon visibility set to Visible
```

**Root Cause:**
1. MainWindow constructor subscribes to Loaded event
2. OnWindowLoaded calls SetupTrayIcon (first call)
3. App.xaml.cs shows MainWindow which triggers Loaded event again
4. SetupTrayIcon called second time, creating duplicate icon

**Fix Applied:**
- Added _isTrayIconInitialized boolean flag
- SetupTrayIcon returns early if already initialized

**Files Modified:**
- src/Redball.UI.WPF/Views/MainWindow.xaml.cs
