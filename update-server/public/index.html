Key Features

Everything You Need, Nothing You Don't

Redball packs powerful clipboard automation and system management into a lightweight Windows app.

⌨️

TypeThing — Clipboard Typer

Types clipboard content character-by-character using the Windows SendInput API with KEYEVENTF_UNICODE. Works in apps that block Ctrl+V — remote desktops, secure terminals, VMs, and web forms.

💤

Smart Keep-Awake

Prevents Windows from sleeping using SetThreadExecutionState with configurable heartbeat keypresses (F13–F16). Battery-aware, network-aware, idle detection, and scheduled operation built in.

📊

Built-in Analytics

Session tracking, feature adoption metrics, and usage statistics viewable in an built-in analytics dashboard. Export data to CSV or JSON at any time.

🎨

14 Custom Themes

Dark, Light, Midnight Blue, Forest Green, Ocean Blue, Sunset Orange, Royal Purple, Slate Gray, Rose Gold, Cyberpunk, Coffee, Arctic Frost, High Contrast, and auto-detect System theme.

🔄

Delta Sync Updates

Binary delta patching downloads only changed bytes between releases, significantly reducing update sizes. Auto-update checks in the background with configurable intervals.

🔒

Code Signed & Secure

EXE and installer signed with SHA-256 certificate via signtool in CI. VirusTotal scanned on every release. Tamper detection and Windows Credential Manager integration.

Deep Dive

Powerful Tools, Zero Complexity

Explore Redball's core capabilities — each built as a native C# service with no external dependencies.

TypeThing — Clipboard Typer

The flagship feature of Redball. TypeThing reads text from your clipboard and types it character-by-character using the Windows SendInput API with KEYEVENTF_UNICODE, making it compatible with virtually any application and character set.

  • Human-like typing with random delays (10–500ms)
  • Global hotkeys: Ctrl+Shift+V to start, Ctrl+Shift+X to stop
  • Full Unicode support via KEYEVENTF_UNICODE
  • Optional text-to-speech while typing

Keep-Awake — System Monitor

Prevents Windows from sleeping using the SetThreadExecutionState API with configurable heartbeat keypresses. Includes intelligent monitoring that reacts to system conditions automatically.

  • Timed sessions (15, 30, 60, 120 min or indefinite)
  • Invisible F13–F16 heartbeat keypresses
  • Display sleep control option
  • Toast notification system

Smart Monitoring & Automation

Redball automatically adapts to system conditions with 10+ built-in monitoring services, all configurable from the settings UI or JSON config file.

  • Battery-aware mode (auto-pause on low charge)
  • Network-aware, idle detection, VPN detection
  • Scheduled operation (start/stop times & days)
  • Thermal protection & process watcher

Our Story

Built by Engineers,
for Engineers

Redball started as a simple internal tool to solve a common frustration: applications that block clipboard paste. The TypeThing feature was born — typing clipboard content character-by-character using the Windows SendInput API.

Today, Redball is a full-featured .NET 10 WPF desktop application with 40+ services, 14 themes, smart keep-awake monitoring, a plugin system, analytics, and code-signed releases. Open source under the MIT license.

View on GitHub
Mar 2026 — Inception
Project Foundation
Initial commit and KMDF driver development. Built build automation, version management, and HID input filter driver foundation.
Mar 2026 — WPF Birth
Native WPF Application
Complete WPF UI with 14 themes, MVVM architecture, TypeThing engine, and P/Invoke SendInput. Reduced EXE from 150MB to 69MB with compression.
Mar 2026 — Services
Service Mode & Analytics
TypeThing Service input method for elevated environments. Analytics dashboard with session tracking, feature usage statistics, and 60+ singleton services.

Package Manager Era
Delta update system, Winget/Scoop/Chocolatey publishing, NSIS installer with .NET 10 runtime embedding, and automated CI/CD releases.
