  SECTION A — ARCHITECTURE & CODE QUALITY (1–15)
================================================================================

1.  [M] INTRODUCE DEPENDENCY INJECTION CONTAINER
    Currently services use static singletons (Lazy<T>). Replace with a proper
    DI container (Microsoft.Extensions.DependencyInjection) to improve
    testability, lifetime management, and decoupling.
    Affected: App.xaml.cs, all *Service.cs files, MainViewModel.cs

2.  [M] EXTRACT INTERFACES FOR ALL SERVICES
    None of the services implement interfaces (IKeepAwakeService, IConfigService,
    etc.). Adding interfaces enables mocking in tests and swapping
    implementations. Currently tests have to work around concrete singletons.
    Affected: Services/*.cs, tests/*.cs

3.  [S] IMPLEMENT IDisposable ON AnalyticsService PROPERLY
    AnalyticsService has a Dispose() method but does not implement IDisposable
    interface. The _flushTimer and _lock (ReaderWriterLockSlim) should be
    disposed. App.xaml.cs never calls Dispose on _analytics.
    Affected: AnalyticsService.cs, App.xaml.cs

4.  [S] IMPLEMENT IDisposable ON HealthCheckService
    HealthCheckService has a Dispose() method but does not implement
    IDisposable. The internal HttpClient is never disposed.
    Affected: HealthCheckService.cs

5.  [M] SEPARATE RedballConfig FROM ConfigService.cs
    The RedballConfig class, NotificationMode enum, and HeartbeatInputMode enum
    are all defined inside ConfigService.cs (438 lines). Move models to a
    dedicated Models/ folder for cleaner separation of concerns.
    Affected: ConfigService.cs → Models/RedballConfig.cs, Models/Enums.cs

6.  [M] REDUCE MainWindow.xaml.cs GOD-CLASS (60,910 BYTES)
    MainWindow.xaml.cs is enormous. Extract tray icon logic, TypeThing
    orchestration, keyboard navigation, and notification handling into dedicated
    helper classes or partial classes to improve maintainability.
    Affected: MainWindow.xaml.cs → TrayIconManager.cs, TypeThingController.cs

7.  [M] REDUCE MainWindow.xaml SIZE (61,986 BYTES)
    MainWindow.xaml is extremely large. Break it into UserControls for each
    navigation section (Home, Analytics, Metrics, Diagnostics, Settings, etc.)
    so each view is its own reusable, testable component.
    Affected: MainWindow.xaml → Views/HomeView.xaml, Views/AnalyticsView.xaml, etc.

8.  [S] FIX STATIC HttpClient HEADER MUTATION IN UpdateService
    GetAllReleasesAsync() adds "User-Agent" to DefaultRequestHeaders on every
    call. This accumulates duplicate headers over time and is not thread-safe.
    Set headers once in the static constructor or use HttpRequestMessage.
    Affected: UpdateService.cs line 213

9.  [S] REPLACE DispatcherTimer WITH System.Timers.Timer FOR NON-UI WORK
    KeepAwakeService uses DispatcherTimer which forces all heartbeat and
    monitoring work onto the UI thread. Use System.Timers.Timer or
    PeriodicTimer for background work and marshal to UI only when needed.
    Affected: KeepAwakeService.cs

10. [S] FIX POTENTIAL VARIABLE SHADOWING IN UPDATE SCRIPT
    CreateUpdateScript() in UpdateService.cs declares $targetDir twice in the
    generated PowerShell script (line 424 and reused in the ForEach loop at
    line 439), which will overwrite the outer variable and break the copy logic.
    Affected: UpdateService.cs CreateUpdateScript()

11. [M] ADOPT CommunityToolkit.Mvvm FOR VIEWMODEL BOILERPLATE
    MainViewModel implements INotifyPropertyChanged manually with RelayCommand.
    Using CommunityToolkit.Mvvm provides [ObservableProperty], [RelayCommand]
    source generators, reducing boilerplate significantly.
    Affected: ViewModels/*.cs, Redball.UI.WPF.csproj

12. [S] CONSOLIDATE DUPLICATE NOTIFICATION/LOGGING CODE
    UpdateService has its own Log() method that writes to a separate file
    (Redball.UI.log) instead of using the centralised Logger class. Unify
    all logging through Logger.
    Affected: UpdateService.cs Log() method

13. [M] ADD CANCELLATIONTOKEN SUPPORT TO ASYNC OPERATIONS
    UpdateService.CheckForUpdateAsync() and DownloadAndInstallAsync() do not
    accept CancellationTokens. Users cannot cancel a slow download.
    Affected: UpdateService.cs, AboutWindow.xaml.cs

14. [S] NORMALISE CONFIG PATH RESOLUTION
    ConfigService.DefaultConfigPath navigates 5 levels up ("../../../../..").
    This is fragile and will break if the build output structure changes.
    Use a more robust resolution strategy.
    Affected: ConfigService.cs line 23

15. [S] ADD NULL-CHECK ON Logger BEFORE FIRST INITIALIZATION
    Several services call Logger.Verbose/Debug in their constructors, which
    could run before Logger.Initialize() in App.xaml.cs if instantiation
    order changes. Add a guard or lazy-init pattern to Logger.
    Affected: Logger.cs, ConfigService.cs, KeepAwakeService.cs
